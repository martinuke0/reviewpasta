  <behavior>
    POST /api/generate-review accepts:
    - businessName (required)
    - location (optional)
    - description (optional)
    - stars (required, 1-5)
    - language (optional, 'en' or 'ro', defaults to 'en')

    Returns:
    - Success: { review: "generated review text" }
    - Validation error: { error: "message" } with 400 status
    - API error: Falls back to template, still returns { review: "template text" }

    Test cases:
    1. Valid request with all fields → returns AI review
    2. Missing required fields → returns 400 error
    3. Invalid stars (0, 6) → returns 400 error
    4. Invalid language → defaults to 'en'
    5. OpenRouter API failure → returns template fallback
    6. No API key configured → returns template
  </behavior>

    - If no key or API fails: use template generation
    - Return consistent { review: string } response
